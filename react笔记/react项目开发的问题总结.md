#关于react开发过程中遇到的问题
```
    1、关于多个模块的样式问题
        webpack打包编译时，所有模块的css或者less，会经过编译打包到一起，因此，在每个样式类名的命名的不规范很容易导致样式冲突。

        解决方法：
            1、使用less或者sass预编译语言，每个版块最外层的样式类名保证唯一性(命名规范：板块名+box等修饰词)，把当前版块下的子内容都嵌套到这个里边，需要公用的样式可以放置在css/common.css下.
            2、使用webpack的插件来打包时进行版块区分。
            
    2、真是项目中并不算是所有的组件都和redux有关系，所以对于那些和redux没有关系的组件，我们不要在使用react-redux中的connect来处理组件(同理对于不需要路由中的属性的一些组件，也没必要使用withRouter来处理)，也就是尽量少的使用高阶组件，因为高阶组件都是利用了柯理化函数思想，形成闭包，这样导致很多栈内存不被销毁，影响性能)

    3、在react中使用redux-promise中间件的时候，action中的函数返回的对象即传递给reducer的action数据（注意：这个action模块中每个方法返回的对象的数据，一般是一些异步的方法，即从服务器获取数据，开始的时候是返回promise）中的属性名必须为payload，即返回的对象除了type，另一个参数名必须是payload名（严格区分大小写）的属性，只有这样promise成功的时候，中间件才会帮助我们重新的dispatch派发一次给reducer，然后把获取的数据更新到reducer容器中的状态中，然后渲染视图。
        解决方法：
            使用react-thunk中间件。
```
