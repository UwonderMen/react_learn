#关于react数据绑定方案
```
    react真实项目中，完成数据绑定，我们可以按照以下方案处理：
        1、第一次加载组件之前或者之后(即生命周期函数componentWillMount或者componentDidMount)，发送ajax请求，等待数据请求成功后，把请求回来的数据更新到组件内部的状态信息（第一次加载的是空数据，第二次更新的时候加载的是真实的数据）从而渲染组件，展示真实的内容
            弊端：
                在路由切换的时候，当前组件很有可能需要重新加载（组件完成了从页面移除到再次展示的过程，这样需要过完组件的整个声明周期流程即从constructor构建组件实例开始到组件声明周期结束这一流程），这样就导致只要路由切换到这个组件，都需要重新发一次ajax请求，对于一些不是随时更新数据的组件，这样处理会增加http请求次数，同时增加服务器负担

        2、每一次加载组件，我们首先验证redux中是否存储了展示的信息，如果有，直接从redux获取即可，如果没有，则发送一个dispatch派发，在怕发的action中基于ajax获取数据请求，把获取的信息传递给reducer，即把信息存储到redux中，一旦redux中的信息更改，那么用到redux中相关的状态信息的组件，在状态更新时，组件都会重新渲染
            弊端：
                某些特定的案例中会存在一些问题，需要做一些额外的处理。例如（在个人中心中，a用户登录成功后，进入个人页面，首先会把a的信息存储到redux中，这样进入信息页面，展示信息都是a用户的，即使你切换，都会从redux中获取，不会从服务器拉取数据，即使a用户的信息更改后，也不会去服务器获取，或者是此时b用户登录了，都不会获取b的信息（主要是我们做了缓存处理，即把旧数据保存起来了（如果有着份数据，就不去获取新数据））。因此，对于这种特殊情况我们需要做额外处理（例如：在重新登录时或者修改用户信息时或者退出登录时都去从服务器获取最新信息）。）
```
