#组件的分类
```
    1、函数式组件
    2、类定义式组件
```

##函数式声明
```
    定义一个函数，里边返回一个jsx元素
    注意：
        不管类定义式组价函数函数式定义的组件，都要引入react模块，否则报错
    *要想在其他组件中使用这个组件，必须导出当前组件*

    函数式组件声明语法：
        import React from "react"
        export default function Vote(){
            return <div>21</div>
        }
    
    函数式组件在调取时候做了哪些事？
        调取组件的时候，babel解析器将其解析后，传递给React.createElement的第一个参数type不是字符串标签名，而是一个函数或者类，即通过React.createElement()方法生成的对象，对象中的type是一个函数或者类。当调用ReactDOM.render()方法的渲染的时候，会根据React.createElement()方法生成的对象中的type类型做不同的事情(如果是字符串就创建新元素，如果是函数，就把函数执行，函数执行的时候会把解析出来的对象中的props作为参数传递给执行的函数，把返回的jsx对象创建为新的元素进行渲染)。
```

#函数式组件的特点
```
    1、函数式组件会把基于React.createElement()方法解析出来的对象中的props作为参数传递给组件，可以完成多次组件传递不同信息
    2、函数是组件调取成功后，返回的jsx就会渲染到页面上，但是后期不通过重新调取组件或者获取dom元素操作的方式，很难再把渲染好的组件中的内容更改。

    注意：函数式组件是静态组件
```

#类定义式组件
```
    类定义式组件，也可以说成继承式组件
    类定义式组件的语法：
        export default class Vote extends React.Component{
            constructor(){
                super()  --->相当于React.Component.call(this)
            }
            render(){
                return <div></div>
            }
        }
    注意：
        1、类定义式组件中必须定义一个render()方法，且返回一个jsx元素。
        2、类定义式组件中的constructor方法中调用了super()方法，是为了继承父类的一些私有属性，比如：可以继承props、context、updater、refs等私有属性。

```

#类定义式组件中关于props中的特点
```
    在类定义式组件中即使我们没有显示将props挂在到组件实例上。但是也可以使用，因为react在实例化组件的时候已经帮我们做了这件事。当然我们也可以显示的将props挂在到实例上，通过constructor()方法中的super()方法进行显示挂载。
```

#类定义式组件中状态的初始化
```
    在类定义式组件中可以使用状态管理，通过在constructor()方法中是声明一个状态初始化结构就行，如下：
        this.state = {
            状态名:初始化状态值
        }
    我们也可以基于this.setState()方法修改状态并且通知组件重新渲染（这个方法是一个异步操作，如果后边有代码，先执行其他代码，然后再去通知状态修改。因此可以看到这个方法有一个回调函数），这个方法支持两个参数：
        第一个参数：修改的部分状态
        第二个参数：通知页面渲染，渲染之后执行的回调函数(一般不用)
```

#类定义式组件特点
```
    1、调取组件相当于创建实例,把一些私有的属性挂在到实例上，这样组件内容所有的方法就可以基于实例获取到这些值(包括传递的属性this.props和状态this.state)
    2、有自己的状态管理，当状态发生改变的时候，react会重新渲染视图(只渲染更改的部分内容，基于dom-diff算法实现)

```


#函数式组件与类定义式组件区别
```
    1、函数式组件调取的时候，直接将这函数执行，将返回回来的jsx通过React.createElement()方法转化成一个对象后，再将其挂载到相应的节点上。
    2、而类定义式组件在调取时，先初始化一个类，将获取的一些props属性作为实例化的参数传递给组件进行是实例化。实例化后，执行实例的render()方法，将返回的jsx元素通过React.createElement()方法转化成一个对象后,再将其挂在到相应节点上。
```

#react也提供了给我们操作dom的方法---this.refs
```
    我们可以给指定的元素上面增加一个ref属性，然后就可以在组件实例中使用this.refs获取指定元素上的ref属性对应的值的DOM元素
```
