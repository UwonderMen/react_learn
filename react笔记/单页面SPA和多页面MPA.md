
#单页面应用和多页面应用
```
    单页面应用又叫SPA（sigle page application）
        一个项目只有一个页面组成，所有的需要展示的内容都在这一个页面中进行切换,如果基于webpack打包编译，那么只需配置一个入口。移动端或者管理类系统单页面居多

    多页面应用又叫MPA（multi page application）
        一个项目由很多页面组成，使用这个产品，主要就是页面之间的跳转。基于框架开发的时候，如果说使用了webpack打包工具，需要配置webpack多入口文件，每个入口对应一个页面。
```

#如何实现单页面应用？
```
    1、如果项目是基于服务器渲染的，后台语言可以基于include等技术，把很多部分内容拼凑在一起，实现组件化或者插件开发，也可以实现单页面应用
    2、基于iframe实现单页面，主要是通过js来改变iframe的src属性
    3、基于模块化开发
        AMD：require.js
        CMD：sea.js
        基于这些思想，把每一部分单独写成一个模块，最后基于这些grunt/gulp/fis等自动化构建工具，最后把所有的模块都合并到首页(包括html、css、js都合并在一起)，通过控制那些模块的显示隐藏来实现单页面应用开发
    
        这种开发存在的弊端：
            首屏渲染慢的问题。即由于首页中的内容包含所有模块的信息，所以第一次加兹安速度很慢。
    
    4、基于vue或者react实现模块化组件开发，基于他们提供的路由实现SPA单页面应用

```
